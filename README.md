This project demonstrates how to create a 3D assembly by importing multiple STL parts and combining them into a single model using build123d in Python. The workflow begins by importing the required library and loading the individual STL files Bottom.stl, Top.stl, and Top-holes.stl, where each file represents a separate component of the model. The import_stl() function loads these STL files as 3D mesh objects in build123d, and each STL file represents a closed triangulated mesh that defines a physical volume in space. Since these parts already contain their own geometric volumes, the script works entirely with solid 3D geometry (volumetric data) rather than sketches or surface features. After importing the parts, they are combined into a single assembly using the Compound() function, which groups multiple 3D objects together without altering their original geometry. The final assembled model is then exported as a new STL file named garry_assembly.stl, and a confirmation message is printed to indicate that the assembly has been successfully generated. This workflow therefore creates a volumetric 3D assembly composed of multiple solid bodies.
The script contains no symmetry operations. In CAD modeling, symmetry operations typically include transformations such as mirror, rotate-copy, linear pattern, or circular pattern. However, this workflow only performs three steps: importing STL files, grouping them using Compound(), and exporting the final assembly. Since no functions such as mirror(), rotate(), or pattern-based transformations from build123d are used, the number of symmetry operations is 0. Mathematically, if the imported parts have volumes V1,V2 & V3 the final assembly can be expressed as Assembly= {V1.V2,V3} which represents a simple grouping of volumes without any geometric transformation.
